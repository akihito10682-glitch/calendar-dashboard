# 📅 カレンダーダッシュボード

Googleカレンダーと連携した、シンプルで見やすいダッシュボードWebアプリです。

![スクリーンショット](https://via.placeholder.com/400x800/667eea/ffffff?text=Calendar+Dashboard)

## ✨ 特徴

- 📱 **スマホ最適化**: 縦長画面向けの見やすいレイアウト
- 🕐 **リアルタイム更新**: 時刻と時限が自動更新
- 📍 **進行中の予定を強調表示**: 現在実行中の予定が一目でわかる
- ⏭️ **次の予定にフォーカス**: 次に何をすべきかすぐわかる
- 🏫 **学校の時間割対応**: カスタマイズ可能な時限表示
- 🔄 **簡単リフレッシュ**: ボタン一つで予定を再読み込み

## 📋 表示内容

### ヘッダー
- 今日の日付
- 現在時刻（リアルタイム）
- 現在の時限（SHR、1限〜6限、昼休み等）
- 次の時限までの残り時間

### メインエリア
- **進行中の予定**: 現在実行中の予定（プログレスバー付き）
- **次の予定**: 最も近い未来の予定を大きく表示
- **今日の予定リスト**: 本日の全予定を時系列で表示

## 🕐 時間割設定

デフォルトの時間割:

| 時限 | 開始 | 終了 |
|------|------|------|
| SHR | 8:40 | 8:50 |
| 1限 | 8:55 | 9:45 |
| 2限 | 9:55 | 10:45 |
| 3限 | 10:55 | 11:45 |
| 4限 | 11:55 | 12:45 |
| 昼休み | 12:45 | 13:25 |
| 5限 | 13:25 | 14:15 |
| 6限 | 14:25 | 15:15 |

時間割は `calendar-dashboard.html` の `periods` 配列で変更できます。

## 🚀 セットアップ

詳しい手順は [SETUP_GUIDE.md](SETUP_GUIDE.md) を参照してください。

### 簡単な流れ

1. **Google Cloud Consoleでプロジェクト作成**
   - OAuth 2.0 クライアントID作成
   - APIキー作成
   - Google Calendar API有効化

2. **HTMLファイルを編集**
   - `CLIENT_ID` と `API_KEY` を設定

3. **GitHubにアップロード**
   - 新規リポジトリ作成
   - HTMLファイルをアップロード

4. **GitHub Pages有効化**
   - Settings → Pages → Source: main
   - 数分で公開完了

## 📱 使い方

### 初回使用時
1. 公開されたURLにアクセス
2. 「Googleでサインイン」ボタンをクリック
3. Googleアカウントでログイン
4. カレンダーへのアクセスを許可

### 日常使用
- ページを開くだけで今日の予定を確認
- 🔄ボタンで予定を手動更新
- スマホのホーム画面に追加すればアプリのように使える

## 🎨 カスタマイズ

### 色の変更

`<style>` タグ内で以下の色を変更できます:

```css
/* メインカラー */
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);

/* アクセントカラー */
color: #667eea;
```

### 時間割の変更

`periods` 配列を編集:

```javascript
const periods = [
    { name: 'SHR', start: { h: 8, m: 40 }, end: { h: 8, m: 50 } },
    // ... 追加・変更
];
```

## 🔒 セキュリティ

- **APIキーの制限を推奨**: HTTPリファラーで制限設定
- **OAuth認証**: Googleの安全な認証システム使用
- **読み取り専用**: カレンダーの閲覧のみ（編集不可）

## 🐛 トラブルシューティング

### 認証エラーが出る
- クライアントIDとAPIキーを再確認
- OAuth設定のURLが正しいか確認
- ブラウザのキャッシュをクリア

### カレンダーが表示されない
- Google Calendar APIが有効化されているか確認
- ブラウザの開発者ツール（F12）でエラー確認
